## Правило адаптивности: мобилка отдельно от десктопа

### КРИТИЧЕСКИ ВАЖНО
При исправлении мобильной версии ЗАПРЕЩЕНО менять стили десктопной версии.

### Как работает Tailwind responsive
- Стили БЕЗ префикса = мобилка (базовые стили, ширина < 768px)
- `md:` = планшет/десктоп (768px+)
- `lg:` = большой десктоп (1024px+)

### Правило "только мобилка"
Когда задача связана с мобильной версией:

1. **Меняй ТОЛЬКО базовые стили** (без префикса) или добавляй стили с `sm:` (640px+)
2. **НЕ ТРОГАЙ стили с `md:`, `lg:`, `xl:`** — это десктоп
3. **Если нужно переопределить** — добавляй мобильный стиль И сохраняй десктопный:
   
   НЕПРАВИЛЬНО (сломает десктоп):
   ```
   className="text-2xl p-4"  // было text-4xl p-8
   ```
   
   ПРАВИЛЬНО (мобилка маленькая, десктоп остался):
   ```
   className="text-2xl md:text-4xl p-4 md:p-8"
   ```

4. **Перед каждым изменением** проверь:
   - Есть ли уже `md:` или `lg:` версия этого стиля?
   - Если да — не трогай её, меняй только базовый стиль
   - Если нет — добавь `md:` со старым значением чтобы сохранить десктоп

### Паттерны для мобильной версии VoiceMap

Иконки:
```
className="w-6 h-6 md:w-10 md:h-10"     // маленькие на мобилке, большие на десктопе
```

Заголовки:
```
className="text-2xl md:text-5xl font-bold" // меньше на мобилке
```

Отступы:
```
className="p-4 md:p-10"                    // компактнее на мобилке
```

Сетка:
```
className="grid grid-cols-1 md:grid-cols-12 gap-4 md:gap-8"
```

Скрыть на мобилке:
```
className="hidden md:block"                // видно только на десктопе
```

Показать только на мобилке:
```
className="block md:hidden"                // видно только на мобилке
```

### Чек-лист после изменений мобильной версии
1. Открой DevTools → Toggle Device → iPhone SE (375px) → всё выглядит хорошо?
2. Открой на полную ширину → десктоп НЕ изменился?
3. Проверь промежуточную ширину (768px) → нет ли "сломанного" состояния?
